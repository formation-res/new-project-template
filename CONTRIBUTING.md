# Contributing

1. Create a focused branch.
2. Keep user-visible text in `public/lang/en-US.ftl`.
3. Run `npm run check`; run `npm run test:e2e` for visible behavior.
4. Include screenshots for visual changes.
5. Explain reusable architectural decisions in `docs/decisions/`.
6. Open a pull request and wait for CI before merging.

Promote behavior into a versioned shared package or reusable workflow when existing projects must continue receiving it.
